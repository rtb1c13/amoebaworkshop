Participating in the workshop at Southampton

Side notes

Occasionally throughout the exercises you'll see boxes like this one, and references to 'side notes' in the text. These notes provide extra information or background on an AMOEBA topic, but aren't necessary for the workshop activities to run. You can read them in your own time during the workshop, or afterwards. Please feel free to ask questions about them too!
Hello! Welcome to this workshop on performing molecular simulations with AMOEBA. These pages contain exercises that go through the basics of using the AMOEBA (Atomic Multipole Optimised Energetics for Biomolecular Application) in the Tinker package. This workshop consists of exercises that will go through the basics of running simulations in Tinker, how to newly parameterise small molecules (such as ligands in protein-ligand interactions) consistently with rest of the AMOEBA forcefield, and how to set up and analyse an example solvation free energy calculation for a small molecule in a box of water.

There will also be brief talks introducing and summarising each exercise, and a research talk giving an overview of some current applications of AMOEBA, including its implementation in other software packages and hardware, such as OpenMM and GPUs.

Slides from the workshop are available here.

The exercises in this workshop are based on those written by Paul Nerenberg of Cal State - if you wish to find more examples of AMOEBA & Tinker tutorials, visit his workshop site.

Getting started

We'll be running our simulations remotely using the Lyceum teaching cluster at Southampton. You should have been provided with login credentials to the cluster on arrival today. The Windows machines we're using have an ssh client, scp interface and VMD pre-installed to log in to remote machines, copy files to and fro, and visualise molecular structures and trajectories.

To log in onto Lyceum, start the PuTTY SSH client by clicking:

START -> ALL PROGRAMS -> Internet and Network Access -> PuTTY -> PuTTY

You'll be presented with a screen like the one on the left below. In the field 'hostname' type 'lyceum2', and click 'Open' at the bottom of the window (the rest of the default options are all fine). Select 'yes' for any of the security messages that pop up, then enter your provided temporary username and password when prompted - see the image on the right below:

Note: For everything except eduroam wifi access, you do NOT need to include the "@soton.ac.uk" at the end of your username. Log in below using only the "confXXX" part of your username. Let us know if you have any problems!

You will find yourself logged into your $HOME directory. It should contain the folders:

  • tinker - this is the location of the Tinker source code and executables
  • gdma - this is the location of the GDMA source code and executables (used in Exercise 2)
  • amoebaworkshop - this contains the input & output files used in today's workshop

All of today's activities will be carried out in the amoebaworkshop folder.

After periods of inactivity you may be logged out of Lyceum. If that happens, don't worry - just log in again.

For each exercise, be sure to take time to study the input files before submitting calculations and don’t be afraid to ask questions.

To follow the instructions, you should be able to use a text editor in a terminal. This tutorial assumes you will be using vi, but feel free to use whichever editor you are most comfortable with (e.g. nano, emacs)

Basic vi syntax

open a file:            vi file
insert text:            i [insert mode], then type text
move cursor:            Escape [normal mode], then arrow keys
save:                   Colon [command mode], then w, then return
save and exit:          Colon [command mode], then wq, then return
exit & abandon changes: Colon [command mode], then q!, then return
delete one line:        Escape [normal mode], then dd
undo:                   Escape [normal mode], then u

Transferring files to and from Lyceum2

At various points throughout the workshop you'll have the option to visualise the biomolecular systems & files you've created. To do so, you'll have to transfer files from Lyceum to the Windows machine you are using. To copy files back from Lyceum to your workstation you can use the WinSCP software, available at:

START -> ALL PROGRAMS -> Your School Software -> Electronics and Computer Science -> WinSCP 5.7.7

Open up the software and you'll see a login screen similar to that for PuTTY. Type 'lyceum2' as the hostname, along with your temporary username and password in the requested boxes:

Click 'Yes' on any security messages that pop up and you'll be presented with a GUI of the directories on your local Windows machine (left) and Lyceum (right). From there it's simply a case of navigating to the file(s) you wish to copy. Files can be transferred by dragging & dropping from one side to the other - you'll then see a 'Download' box like the one below:

For today's workshop we suggest it's easiest to just copy files from Lyceum to the Windows Desktop folder.

VMD

We will typically use VMD to visualise the simulations. To open VMD, navigate to:

START -> ALL PROGRAMS -> Your School Software -> Chemistry -> Visual Molecular Dynamics -> VMD 1.9.2

Movies and files can be loaded into VMD by clicking "File | New Molecule...", and then click "Browse..." in the pop up window to find the ".pdb" file that you downloaded. Make sure to then select 'Tinker' in the file type menu, as VMD by default interpret the xyz file extension as a different file type:


Then click "Load" to load the movie/file. If it's a movie, press the play button at the bottom of the "VMD Main" window to get it going.

Running the workshop elsewhere

Software & hardware requirements

This workshop was written to be run on the Lyceum teaching cluster at the University of Southampton. If you're running it elsewhere, the following software is what we used (previous software versions may also work, but we make no guarantees). We also recommend access to a desktop or cluster with at least 8 cores per node for running the simulations:

  • Tinker 7.1.2 or later
  • Intel or Gnu Fortran compilers
  • Gaussian 03 or later
  • GDMA 2.2.11
  • VMD 1.9.1 or later
